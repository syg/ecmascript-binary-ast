    Binary AST

    The binary AST is isomorphic to ECMA-262 Syntactic Grammar. That is, an instance of a production of the binary AST grammar, or a binary AST Node, corresponds to some Parse Node in the ECMA-262 syntactic grammar. This document specifies the following things.

    1. The tree grammar itself described in IDL and its well-formedness.
    2. The transformation abstract operation Ecmaify(binaryAST) from binary AST Nodes to Parse Nodes by case analysis.
    3. TODO: The similarity Static Semantics ParseNodesAreSimilar.
    4. The evaluation of binary AST input.

    A compliant binary AST Encoder, for a source text that matches either the Script or Module goal symbol to produce a Parse Node N, must produce a binary AST Node B such that ParseNodesAreSimilar(Ecmaify(B), N) is true.

    The inverse of Ecmaify, Unecmaify, is not specified here. The inverse transform is the encoding of Parse Nodes. For B and N as above, for all M such that ParseNodesAreSimilar(M, N) is true, Unecmaify(M) is B.

    This section is derived from Shift AST Spec and parts are Copyright 2014-2017 Shape Security, Inc.

    Unlike the Shift AST spec, interface types are not used to inherit fields to control over ordering of fields. Type hierarchies that are not used to discriminate are collapsed to make space costs simple. Nor are they used to discriminate types, for which explicitly discriminated unions types are used.

    Note
    Whereas Shift AST's design principle is ease of search-and-replace of node types, binary AST's design principle is ease of verification and ease of associating different behaviors with syntactically different (but possibly lexically similar) productions.

    The grammar is presented in WebIDL with the [TypeIndicator] and NonEmpty extensions per Shift AST spec. The typedefs of or types are to be read as recursive sum types. In text below, the "is a Foo" prose is shorthand for checking the node's type attribute being equal to "Foo".

    2Transformation of AST to Parse Tree

    The Ecmaify abstract operation transforms binary AST Nodes to ECMAScript Parse Nodes. It performs additional early error checking on top of existing ECMAScript early error semantics.

    A Parse Node may be tagged as being transformed by a binary AST Node. The original binary AST Node is retrievable from tagged Parse Nodes.

    Note
    Tagging Parse Nodes is used to introduce laziness at function body boundaries.

    The naming convention of the Ecmaify abstract operations is:

    • FooEcmaify always returns a Parse Node of the ECMAScript grammar production Foo.
    • EcmaifyBar only takes binary AST Node with interface Bar and returns Parse Nodes of varying ECMAScript grammar productions.
    • Ecmaify is the central dispatcher that dispatches on the interface type of its input.

    2.1HasMalformedDirectives ( directives, nodes )

    1. Assert: directives is a FrozenArray<Directive>.
    2. Assert: nodes is a FrozenArray<Node>.
    3. For each node in nodes, do
      1. If the node is an ExpressionStatement and node.expression is a StringLiteralExpression:
        1. For each dir in directives, do
          1. If node.expression.value is the same value as dir.rawValue, then return true.
        2. NOTE: This restriction disallows encoding directives as string literals.
    4. Return false.

    2.2StatementEcmaify ( stmt )

    1. Assert: stmt is a Statement.
    2. Let n be ? Ecmaify(stmt).
    3. If stmt is a Block, then
      1. Set n to BlockStatement : n.
    4. Else if stmt is an IterationStatement, then
      1. Set n to BreakableStatement : n.
    5. Else return Statement : n.

    2.3StatementListEcmaify ( stmts )

    1. Assert: stmts is a FrozenArray<Statement>.
    2. Let list be an empty Parse Node.
    3. If stmts has length 0, then
      1. Let emptyStmt be StatementListItem : EmptyStatement.
      2. Set list to StatementList : emptyStmt.
    4. For each stmt in stmts, do
      1. If stmt is a FunctionDeclaration, then
        1. Set n to HoistableDeclaration: ? Ecmaify(stmt).
        2. Set n to be Declaration : n.
      2. Else set n to be ? StatementEcmaify(stmt).
      3. Set n to be StatementListItem : n.
      4. If list is empty, then set list to StatementList : n.
      5. Else set list to StatementList : list n.
    5. Return list.

    2.4VariableDeclarationListEcmaify ( decls )

    1. Assert: decls is a FrozenArray<VariableDeclarator>.
    2. Assert: decls does not have length 0.
    3. Let list be an empty Parse Node.
    4. For each decl in decls, do
      1. Let n be an empty Parse Node.
      2. Let binding be ? Ecmaify(decl.binding).
      3. If decl.init is null, then
        1. If decl.binding is not a BindingIdentifier, throw a SyntaxError exception.
        2. Set n to VariableDeclaration : binding.
      4. Else,
        1. Let init be ? Ecmaify(decl.init).
        2. Set n to VariableDeclaration : binding init.
      5. If list is empty, set list to VariableDeclarationList : n.
      6. Else set list to VariableDeclarationList : list n.
    5. Return list.

    2.5BindingListEcmaify ( decls )

    1. Assert: decls is a FrozenArray<VariableDeclarator>.
    2. Assert: decls does not have length 0.
    3. Let list be an empty Parse Node.
    4. For each decl in decls, do
      1. Let n be an empty Parse Node.
      2. Let binding be ? Ecmaify(decl.binding).
      3. If decl.init is null, then
        1. If decl.binding is not a BindingIdentifier, throw a SyntaxError exception.
        2. Set n to LexicalBinding : binding.
      4. Else,
        1. Let init be ? Ecmaify(decl.init).
        2. Set n to LexicalBinding : binding init.
      5. If list is empty, set list to BindingList : n.
      6. Else set list to BindingList : list n.
    5. Return list.

    2.6ElementListEcmaify ( elems )

    1. Assert: elems is a FrozenArray<SpreadElement or Expression>.
    2. Assert: elems does not have length 0.
    3. Let list be an empty Parse Node.
    4. Let accumulatedElisions be an empty Parse Node.
    5. For each elem in elems, do
      1. Let n be an empty Parse Node.
      2. If elem is null, then
        1. If accumulatedElisions is empty, then set accumulatedElisions to Elision : ,.
        2. Else set accumulatedElisions to Elision : accumulatedElisions ,.
      3. Else,
        1. If elem is a SpreadElement, then
          1. Let expr be ? AssigmentExpressionEcmaify(elem.expression).
          2. Set n to SpreadElement : ... expr.
        2. Else set n to ? AssignmentExpressionEcmaify(elem).
        3. If list is empty, then
          1. If accumulatedElisions is empty, then set list to ElementList : n.
          2. Else set list to ElementList : accumulatedElisions n.
        4. Else,
          1. If accumulatedElisions is empty, then set list to ElementList : list , n.
          2. Else set list to ElementList : list , accumulatedElisions n.
        5. Set accumulatedElisions to an empty Parse Node.
    6. Return list and accumulatedElisions.

    2.7FormalParameterListEcmaify ( params )

    1. Assert: params is a FrozenArray<Parameter>.
    2. Assert: params does not have length 0.
    3. Let list be an empty Parse Node.

    1. Assert: args is an Arguments.
    2. Assert: args does not have length 0.

        1. Let expr be ? AssignmentExpressionEcmaify(arg.expression).
        2. If list is empty, then set list to ArgumentList : ... expr.
        3. Else set list to ArgumentList : list , expr.
      2. Else,
        1. Let expr be ? AssignmentExpressionEcmaify(arg).
        2. If list is empty, then set list to ArgumentList : expr.
        3. Else set list to ArgumentList : list , expr.
    5. Return list.

    5. Return list.

    1. Assert: case is a SwitchCase.
    2. Let expr be ? ExpressionEcmaify(case.test).

    5. Else,
      1. Let stmts be ? StatementListEcmaify(case.consequent).
      2. Set clause to CaseClause : case expr : stmts.

        3Evaluation

        Binary AST is evaluated via the modified ScriptEvaluationJob and TopLevelModuleEvaluationJob, defined below. The binary AST is transformed into an ECMAScript parse tree. The resulting parse tree is then evaluated using the same evaluation semantics as for parse trees resulting from source text, with the following exceptions.

        1. Transformation of binary ASTs to parse trees may throw exceptions in the same phase as early errors.
        2. Evaluation of a binary AST is inherently lazy at the function boundaries. A function's FunctionBody is not transformed until it is invoked. This means Static Semantics which would have been observable (i.e. by throwing exceptions) at parse time of functions evaluated via source text is deferred until invocation if the function is evaluated via binary AST.
        3. AssertedScope contains assertions to be checked during transformation of the current AST or future, nested ASTs. For the latter, OutstandingAssertedScope Records are tracked on the current execution context.

        3.1Assertions

        Evaluation of binary AST is inherently lazy at function boundaries. Together with AssertedScope, the intent is to allow implementations to start execution or code generation without parsing the entire tree. AssertedScope represent assertions that must be checked. A failing assertion throws a SyntaxError exception.

        There are two kinds of assertions in AssertedScope: those that eagerly checked during the transformation of the current tree, and those that are checked lazily as inner functions are transformed in the future.

        Checking lazy assertions is done via a List of OutstandingAssertedScope Records on the execution context. After finishing transformation of a Script, Module, FunctionBody, and Block, its AssertedScope's eager assertions are checked. AssertedScopes that have outstanding lazy assertions at the end of transformation, along with any inner lazy functions in its scope, are packaged in an OutstandingAssertedScope Record and appended to the current execution context. When an inner function is transformed, any outstanding assertions are checked. If assertions remains, any new inner functions seen during the transformation are appended to the OutstandingAssertedScope Record. If all assertions have been satisfied, the OutstandingAssertedScope Record is removed from the execution context. If there are unsatisfied assertions after all inner functions have been transformed, a SyntaxError is thrown.

        Note
        While AssertedScopes are optional, an implementation is prohibited from binding optimizations if they are absent.

        The GlobalOutstandingAssertedScopes is a List that is globally available. It is shared by all realms. Prior to the evaluation of any ECMAScript code it is initialized as a new empty List. Elements of the GlobalOutstandingAssertedScopes are Records with the structured defined in Table 1.

        3.1.1AddGlobalOutstandingAssertedScope ( outstandingAssert )

        1. Assert: outstandingAssert.[[HasDirectEval]] is not "satisfied" or outstandingAssert.[[CapturedNames]] is a non-empty List.
        2. If outstandingAssert.[[InnerFunctions]] is an empty List, throw a SyntaxError exception.
        3. If outstandingAssert is not in GlobalOutstandingAssertedScopes, then
          1. Add outstandingAssert as the last element of GlobalOutstandingAssertedScopes.

        3.1.2CheckDeclaredNames ( expectedDeclared, actualDeclared )

        1. For each dn in actualDeclared, do
          1. If dn is not in expectedDeclared, then throw a SyntaxError exception.
          2. Remove dn from expectedDeclared.
        2. If expectedDeclared has length greater than 0, then throw a SyntaxError exception.

        3.1.3CheckAssertedScope ( scope , parseTree )

        1. Assert: scope is an AssertedBlockScope? or an AssertedVarScope? or an AssertedParameterScope?.
        2. Assert: parseTree is a non-empty Parse Node.
        3. If scope is null, return.
        4. Let declaredNames be a new empty List.
        5. If scope is an AssertedBlockScope or an AssertedVarScope, then
          1. If scope is an AssertedVarScope, then
            1. Append the elements of scope.varDeclaredNames to declaredNames.
          2. Append the elements of scope.lexicallyDeclaredNames to declaredNames.
        6. Else,
          1. Append the elements of scope.parameterNames to declaredNames.
        7. NOTE: All captured names must be asserted bindings in the scope.
        8. For each String cn in scope.capturedNames, do
          1. If cn is not in declaredNames, then throw then a SyntaxError exception.
        9. NOTE: All asserted declared names must be declared.
        10. If scope is an AssertedBlockScope or an AssertedVarScope, then
          1. If scope is an AssertedVarScope, then
            1. Let varDeclaredNames be the VarDeclaredNames of parseTree.
            2. Perform ? CheckDeclaredNames(scope.varDeclaredNames, varDeclaredNames).
          2. Let lexDeclaredNames be the LexicallyDeclaredNames of parseTree.
          3. Perform ? CheckDeclaredNames(scope.lexicallyDeclaredNames, lexDeclaredNames).
        11. Else,
          1. Let paramNames be the BoundNames of parseTree.
          2. Perform ? CheckDeclaredNames(scope.parameterNames, paramNames).
        12. NOTE: Presence of direct eval and veracity of captured names cannot be fully checked until inner functions are evaluated.
        13. Let outstandingCapturedNames be a new empty List.
        14. Let innerFunctions be a new empty List.
        15. For each FunctionBody fb contained in parseTree, do
          1. Add fb as the last element of innerFunctions.
        16. Let outstandingAssert be a new OutstandingAssertedScope Record {[[CapturedName]]: outstandingCapturedNames, [[HasDirectEval]]: "satisfied", [[InnerFunctions]]: innerFunctions}.
        17. Let containsDirectEval be true if parseTree contains a direct eval, false otherwise.
        18. If it is not the case both containsDirectEval and scope.hasDirectEval are true, then
          1. Set outstandingAssert.[[HasDirectEval]] to scope.hasDirectEval.
          2. Perform ? AddGlobalOutstandingAssertedScope(outstandingAssert).
        19. If scope.capturedNames has length great than 0, then
          1. Perform ? AddGlobalOutstandingAssertedScope(outstandingAssert).

        3.1.4DischargeOutstandingAssertedScopes ( funcBody )

        1. Assert: funcBody is a FunctionBody.
        2. Let innerFunctions be a new empty list.
        3. For each FunctionBody fb contained in funcBody, do
          1. Add fb as the last element of innerFunctions.
        4. Let freeNames be the FreeNames of funcBody.
        5. For each OutstandingAssertedScope Record outstandingAssert in GlobalOutstandingAssertedScopes, do
          1. If outstandingAssert.[[InnerFunctions]] contains funcBody, then
            1. Remove funcBody from outstandingAssert.[[InnerFunctions]].
            2. Let allSatisfied be true.
            3. For each fn in freeNames, do
              1. Remove fn, if present, from outstandingAssert.[[CapturedNames]].
            4. If outstandingAssert.[[CapturedNames]] not an empty List, then
              1. Set allSatisfied to false.
            5. If outstandingAssert.[[HasDirectEval]] is not "satisfied", then
              1. Let containsDirectEval be true if funcBody contains a direct eval, false otherwise.
              2. If both containsDirectEval and scope.hasDirectEval are true, then
                1. Set outstandingAssert.[[HasDirectEval]] to "satisfied".
              3. Else,
                1. Set allSatisfied to false.
            6. If allSatisfied is true, then
              1. Remove outstandingAssert from GlobalOutstandingAssertedScopes.
            7. Else,
              1. Remove funcBody from outstandingAssert.[[InnerFunctions]].
              2. Append innerFunctions to the elements of outstandingAssert.[[InnerFunctions]].
              3. If outstandingAssert.[[InnerFunctions]] is an empty List, throw a SyntaxError exception.

        TransformASTScript is a new abstract operation.

        1. Let body be ? EcmaifyScript(ast).
        2. Return Script Record {[[Realm]]: realm, [[Environment]]: undefined, [[ECMAScriptCode]]: body, [[HostDefined]]: hostDefined}.

        3.5ValidateAndUpdateFunctionObject ( funcNode , functionObject, body0, params )

        1. Let prod be an empty Parse Node.
        2. Let body be an empty Parse Node.
        3. If funcNode is a FunctionDeclaration, then
          1. Let funcName be ? Ecmaify(funcNode.name).
          2. If funcNode.isAsync is true, then
            1. Set body toe AsyncFunctionBody : body0.
            2. Set prod AsyncFunctionDeclaration : async function funcName ( params ) { body }.
          3. Else if funcNode.isGenerator is true, then
            1. Set body to GeneratorBody : body0.
            2. Set prod to GeneratorDeclaration : function * funcName ( params ) { body }.
          4. Else,
            1. Set body to body0.
            2. Set prod to FunctionDeclaration : function funcName ( params ) { body }.
        4. Else if funcNode is an ArrowExpression, then
          1. If funcNode.isAsync is true, then
            1. Let asyncHead be AsyncArrowHead : async params.
            2. Let body1 be AsyncFunctionBody : body0.
            3. Set body to AsyncConciseBody : { body1 }.
            4. Set prod to AsyncArrowFunction : asyncHead => body.
          2. Else,
            1. Set body to ConciseBody : { body0 }.
            2. Set prod to ArrowFunction : params => body.
        5. Else if funcNode is a FunctionExpression, then
          1. Let maybeFuncName be an empty Parse Node.
          2. If funcNode.name not null, then set maybeFuncName to ? Ecmaify(funcNode.name).
          3. If funcNode.isAsync is true, then
            1. Set body to AsyncFunctionBody : body0.
            2. Set prod to AsyncFunctionExpression : async function maybeFuncName ( params ) { body }.
          4. Else if funcNode.isGenerator is true, then
            1. Set body to GeneratorBody : body0.
            2. Set prod to GeneratorExpression : function * maybeFuncName ( params ) { body }.
          5. Else,
            1. Set body to body0.
            2. Set prod to FunctionExpression : function maybeFuncName ( params ) { body }.
        6. Else if funcNode is a Method, then
          1. Let name be PropertyName : ? Ecmaify(funcNode.name).
          2. If funcNode.isAsync is true, then
            1. Set body to AsyncFunctionBody : body0.
            2. Let m be AsyncMethod : async name ( params ) { body }.
            3. Set prod to MethodDefinition : m.
          3. Else if funcNode.isGenerator is true, then
            1. Set body to GeneratorBody : body0.
            2. Let m GeneratorMethod : * name ( params ) { body }.
            3. Set prod to MethodDefinition : m.
          4. Else,
            1. Set body to body0.
            2. Set prod to MethodDefinition : name ( params ) { body }.
        7. Else if funcNode is a Getter, then
          1. Let name be PropertyName : ? Ecmaify(funcNode.name).
          2. Set body to body0.
          3. Set prod to MethodDefinition : get name ( ) { body }.
        8. Else,
          1. Assert: funcNode is a Setter.
          2. Let name be PropertyName : ? Ecmaify(funcNode.name).
          3. Set body to body0.
          4. Set prod to MethodDefinition : set name ( param ) { body }.
        9. Validate prod for any Early Error conditions. If early errors were found, then throw a SyntaxError or a ReferenceError exception, depending on the type of the error.
        10. Set functionObject.[[FormalParameters]] to params.
        11. Set functionObject.[[ECMAScriptCode]] to prod.

        3.6Runtime Semantics: Delazify

        Delazify is a new Runtime Semantics.

        With parameter functionObject.

        FunctionBody:FunctionStatementList
        1. If FunctionStatementList is a StatementList consisting of a single EmptyStatement that is tagged as being produced by funcNode, then
          1. Let delazifiedParams be the result of Delazify on the Parse Node that is functionObject.[[FormalParameters]] passing functionObject.
          2. Let delazifiedBody be ? EcmaifyFunctionBody(funcNode.body).
          3. Perform ? ValidateAndUpdateFunctionObject(funcNode, functionObject, delazifiedBody, delazifiedParams).
          4. If funcNode is not a Getter, then
            1. Perform ? CheckAssertedScope(funcNode.parameterScope, delazifiedParams).
          5. Perform ? CheckAssertedScope(funcNode.bodyScope, delazifiedBody).
          6. Perform ? DischargeOutstandingAssertedScopes(delazifiedBody).
          7. Return FunctionBody : delazifiedBody.
        2. Else return FunctionBody.
        FormalParameters:[empty]
        1. If FormalParameters is tagged as being produced by an AST node paramAst, then
          1. Return ? EcmaifyFormalParameters(paramsAst).
        2. Else return FormalParameters.
        UniqueFormalParameters:FormalParameters
        1. Return the result of Delazify of FormalParameters.
        ArrowParameters:CoverParenthesizedExpressionAndArrowParameterList
        1. If ArrowParameters is tagged as being produced by an AST node paramAst, then
          1. Return ? EcmaifyCoverArrowParameterList(paramsAst).
        2. Else return ArrowParameters.
        PropertySetParameterList:FormalParameter
        1. If PropertySetParameterList is tagged as being produced by an AST node paramAst, then
          1. Return ? EcmaifyPropertySetParameterList(paramsAst).
        2. Else return PropertySetParameterList.

        3.7Runtime Semantics: EvaluateBody

        The runtime semantics of FunctionBody is modified as follows.

        With parameters functionObject and List argumentsList.

        FunctionBody:FunctionStatementList
        1. Let delazifiedBody be the Parse Node that is the result of Delazify of FunctionBody.
        2. If delazifiedBody is a different Parse Node from FunctionBody, then
          1. Return the result of EvaluateBody of the Parse Node that is functionObject.[[ECMAScriptCode]] passing functionObject and argumentsList.
        3. Else,
          1. Perform ? FunctionDeclarationInstantiation(functionObject, argumentsList).
          2. Return the result of evaluating FunctionStatementList.
        GeneratorBody:FunctionBody
        1. Let delazifiedBody be the Parse Node that is the result of Delazify of FunctionBody.
        2. If delazifiedBody is a different Parse Node from FunctionBody, then
          1. Return the result of EvaluateBody of the Parse Node that is functionObject.[[ECMAScriptCode]] passing functionObject and argumentsList.
        3. Else,
          1. Perform ? FunctionDeclarationInstantiation(functionObject, argumentsList).
          2. Let G be ? OrdinaryCreateFromConstructor(functionObject, "%GeneratorPrototype%", « [[GeneratorState]], [[GeneratorContext]] »).
          3. Perform GeneratorStart(G, FunctionBody).
          4. Return Completion{[[Type]]: return, [[Value]]: G, [[Target]]: empty}.
        AsyncFunctionBody:FunctionBody
        1. Let delazifiedBody be the Parse Node that is the result of Delazify of FunctionBody.
        2. If delazifiedBody is a different Parse Node from FunctionBody, then
          1. Return the result of EvaluateBody of the Parse Node that is functionObject.[[ECMAScriptCode]] passing functionObject and argumentsList.
        3. Else,
          1. Let promiseCapability be ! NewPromiseCapability(%Promise%).
          2. Let declResult be FunctionDeclarationInstantiation(functionObject, argumentsList).
          3. If declResult is not an abrupt completion, then
            1. Perform ! AsyncFunctionStart(promiseCapability, FunctionBody).
          4. Else declResult is an abrupt completion,
            1. Perform ! Call(promiseCapability.[[Reject]], undefined, «declResult.[[Value]]»).
          5. Return Completion{[[Type]]: return, [[Value]]: promiseCapability.[[Promise]], [[Target]]: empty}.
